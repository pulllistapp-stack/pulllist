## 1. 원칙 (locked)

1. **Free 는 forever free**. 결제 안 해도 결코 잠기지 않음. 커뮤니티 자체가 우리의 moat.
2. **Merchant of Record** 는 **Lemon Squeezy** 를 default. VAT/GST/세일즈세를 MoR가 처리, LO는 개인사업자 간이과세자 단순 소득 신고만.
3. **KR 유저용 원화 결제 (KG이니시스/토스페이먼츠)** 는 **정식 오픈 후 6개월 시점** 재검토. 사업자등록 완료 + Pro MRR ≥ $500 조건 충족 후.
4. **가격은 최대한 심리적 벽 아래에서**. Pokefy $8/월 대비 명확한 언더컷 각도.
5. **연간 할인은 크게**. Retention 지렛대. 20% 이상.
6. **무료 트라이얼 있음**. Pro 가치 체험 없이 결제 요구 안 함.
7. **가격 인상 시 grandfather**. 기존 구독자는 기존 가격 유지 (신뢰 자산).
8. **하드셀 없음**. Pop-up 페이월 · 반복 알림 · 다크 패턴 UI 금지 (`tier-design.md` §6 참조).

## 2. Pro 가격 — 결정 대기 (LO 답 필요)

### Q1. Pro 최종 가격

#### 시장 컨텍스트
| 서비스 | 가격 | 범위 |
|---|---|---|
| **Pokefy** (경쟁 최근접) | $8/월 (알림 온리) | 알림 트랙킹 |
| **PriceCharting Pro** | $50/월 | 프로 셀러/엔터프라이즈 |
| **TCGplayer Pro** | $5/월 | 판매자 대시보드 |
| **ManaBox Premium** | $6/월 | MTG/포켓/등 컬렉션 |
| **Dragon Shield MyDragonShield** | $5/월 | 컬렉션 트래킹 |

#### 옵션
| 옵션 | 월간 | 년간 | 년간 할인율 | 코멘트 |
|---|---|---|---|---|
| **A** — PROJECT_STATUS 원안 | **$5.99** | **$59** | ~18% | 기존 문서 일치, Pokefy 25% 언더컷 |
| **B** — 원안 + 년간 할인 확대 | **$5.99** | **$49** | ~32% | 년간 유입 강화 |
| **C** — $4.99 심리적 벽 밑 | **$4.99** | **$39** | ~35% | 커뮤니티 tier · Pokefy 38% 언더컷 · TCGplayer Pro 동가 |
| **D** — ROADMAP §4 재검토 | **$7.99** | **$69** | ~28% | Pokefy 근접 · 매출 우선 |

**추천: C ($4.99 / $39)**. 이유:
- **$5 심리적 벽** 아래는 결제 저항 크게 낮음 (Steam DLC 심리학과 동일)
- **친구 베타 → 정식 오픈 초기** 는 유료 컨버전 volume 이 매출 metric ; 개당 매출보다 컨버전 rate 가 중요
- **Pokefy 38% 언더컷** 은 SEO / word-of-mouth 에서 명확한 marketing 각도 ("Pokefy 반값에 카탈로그 통짜")
- **$39/년 = 월 $3.25** 실효가격, 년 구독 유입 강한 hook

**보류 팩터**: LO가 Pro tier 를 "진지한 유저에게 파는" 포지셔닝으로 밀 거면 $5.99/$49 (옵션 B) 가 더 정합. 이 경우 커뮤니티 tier + 진지 유저 tier 분할 (§7 미래) 여지 남김.

### Q2. 지역 가격 (PPP)

Lemon Squeezy 는 [manual regional pricing](https://docs.lemonsqueezy.com/help/manage-pricing/regional-pricing) 지원.

- **옵션 A** — 글로벌 단일 가격 (모두 USD)
- **옵션 B** — 아시아 (KR/JP/TW/PH/VN 등) 30% 할인, EU/US 는 기본가
- **옵션 C** — 대륙별 3단 (US $4.99 / 아시아 $3.99 / 남미·동남아 $2.99)

**추천: A (초기 6개월) → B (6개월 후)**. 이유:
- 초기 컨버전 데이터 수집이 우선
- 지역 할인은 유저 지리 통계 나온 후 정확한 targeting
- PP (Purchasing Power) 차별화는 명확한 upsell 아이디어인데, 초기 UX 복잡화 방지

### Q3. 무료 트라이얼

- **옵션 A** — 없음 (Free tier 로 충분)
- **옵션 B** — **7일 트라이얼** (카드 등록 요구)
- **옵션 C** — **14일 트라이얼** (카드 등록 요구)
- **옵션 D** — **7일 트라이얼** (카드 등록 불필요, 이메일 인증만)

**추천: B (7일, 카드 등록 요구)**. 이유:
- 트라이얼 신용 → 결제 완료 컨버전이 카드 요구 시 3-5x 높음 (SaaS 벤치마크)
- 7일은 알림 시스템 hook 이 도는 시간 (한 사이클 = 위시 등록 → 매물 감지 → 알림 수신 → 구매)
- Lemon Squeezy 는 trial 자동 관리 (카드 저장 후 trial 종료 시 자동 결제)

**대안 조합**: 신규 유저 signup 시 자동 트라이얼 시작 vs 유저 명시 opt-in. **추천: opt-in** (명시적 hook 클릭 필요, 실수 결제 방지).

### Q4. 마이크로 결제 (day / week pass)

- **옵션 A** — 없음
- **옵션 B** — Day pass $0.99 · Week pass $2.99
- **옵션 C** — Week pass $2.99 만 (신카드 발매 주 hook)

**추천: A (초기 스킵) → C (6개월 후 재검토)**. 이유:
- 마이크로 결제는 Lemon Squeezy 지원되지만 결제 수수료 (Stripe 30¢ + 2.9%) 가 $0.99에 도달하면 40% 사라짐 = 실효 매출 미미
- 대신 신카드 발매 · 대형 이벤트 (Nationals, 도쿄 카드쇼) 근처 유저 유입은 년간 프로모 (§5) 로 대응

### Q5. 얼리버드 / 런치 프로모

친구 베타 (현재) → 정식 오픈 순간의 하이라이트 순간.

- **옵션 A** — 없음
- **옵션 B** — **초기 100명 Pro 3개월 무료** (`tier-design.md` §4 Q6 추천안)
- **옵션 C** — **초기 100명 Pro lifetime $29** (1회 결제, 영구 Pro)
- **옵션 D** — **초기 1000명 첫해 50% 할인** ($24.50 대신 $19.50/년)
- **옵션 E** — B + D 결합 (100명 3개월 무료 → 다음 900명 첫해 50%)

**추천: E**. 이유:
- 친구 베타 참여 감사 = 특별 대우 (100명 뱃지 · 3개월 무료)
- 다음 900명 (총 1000) 첫해 할인 = 정식 오픈 초기 volume hook
- Lifetime deal (C) 은 위험 — 인프라 비용 (Neon Launch, Render Starter, 이메일 발송) 이 유저당 monthly recurring 인데 lifetime 은 상한 없음

### Q6. 환불 정책

Lemon Squeezy MoR 는 기본 환불 정책 있음 (14일 EU 소비자법 준수).

- **옵션 A** — Lemon Squeezy 기본 (14일 EU / 문의 시 수동 승인)
- **옵션 B** — **7일 무조건 환불** (문의만으로 진행)
- **옵션 C** — **30일 무조건 환불** (강한 신뢰 시그널)

**추천: B (7일 무조건)**. 이유:
- 트라이얼 (Q3) 이 7일이므로 결제 후 유저는 이미 서비스 검증 상태 → 30일 무조건은 오버킬
- 문서화 심플: "7일 안엔 문의만으로 환불, 그 이후는 개별 검토"
- EU 소비자법 14일은 Lemon Squeezy 자동 처리이므로 문서상 7일 얹으면 실효는 14일

### Q7. 학생 · 오픈소스 할인

- **옵션 A** — 없음
- **옵션 B** — 학생 50% 할인 (`.edu` · `.ac.kr` · `.ac.jp` 이메일 인증)
- **옵션 C** — 학생 + 오픈소스 유지자 (GitHub OAuth + N stars) 50%

**추천: A (초기 스킵)**. 이유:
- 구현 복잡도 vs 실제 매출 impact 미미 (컬렉터 인구 중 학생 인증 비율 낮음)
- 정식 오픈 6개월 후 요청이 실제로 들어오면 그때 추가

### Q8. 가격 인상 정책

## 6. 가격 페이지 UX 원칙 (`/pricing` 렌더링 가이드)

`tier-design.md` §5 feature matrix 를 렌더링. 원칙:

1. **월간 가격 큰 폰트, 년간 소폰트로 옆에** ("$4.99/월 · $39/년, 32% 절약")
2. **Free tier 왼쪽 · Pro tier 오른쪽 (하이라이트 그라디언트 border)**
3. **비교 표 하단에 "왜 Pro?" 3-라인 justification** — 알림 실시간 · 무제한 스캔 · 심화 데이터
4. **CTA 버튼**: Free = "무료로 시작" (링크 to /signup) · Pro = "7일 무료 체험" (링크 to Lemon Squeezy checkout)
5. **FAQ 하단**: 환불 / grandfather / 지역 가격 / 트라이얼 자동 결제
6. **하드셀 요소 금지** — 카운트다운 · "N명이 마지막 결제" 팝업 · exit intent 모달 등
